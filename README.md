Smart DTC System 🚌
A journey of a thousand miles (and a few bus stops).
📖 The Story Behind This Project
This wasn't just a coding exercise; it was our entry for the Smart India Hackathon (SIH). My friends and I were incredibly hyped. We cleared the first internal round and the main presentation round at our college with high hopes.

However, things took a turn during the final internal elimination. Just a week before the national PPT submission, our mentor told us that "government-related projects have no scope" and pressured us to change our entire topic. We were devastated but tried to pivot. Because of that last-minute shift and a lack of proper guidance, we ended up submitting messy, half-prepared code just to meet the deadline. Predictably, we didn't make the national cut.

But I didn't want to let the work die there. This repository contains the specific part of the original DTC project that I worked on.

⚠️ A Note on the Code
I’ll be honest: it’s laggy, it’s glitchy, and it’s definitely not the "best" site out there. It was my very first real project, built under extreme pressure and mid-collapse of a hackathon team.

I’m uploading it here as Proof of Work. It shows where I started, the logic I was able to build (even when things were falling apart), and my ability to create a multi-user system from scratch.

🛠️ My Contributions
I handled the frontend architecture and the core logic for the transit system:

Passenger Hub: Created a functional fare calculator and route search engine.

Driver/Admin Panels: Designed the interface for duty management and crew tracking.

Responsive Design: Integrated a dark mode and tried to make it work on mobile (a huge learning curve!).

Validation: Implemented strict input rules for professional data entry.

🚀 Tech Used
Frontend: HTML5, CSS3, JavaScript (ES6)

Icons: Bootstrap Icons

Typography: Google Fonts (Poppins)

💡 What I Learned
Failure is a better teacher than success. This project taught me:

Trust your vision: We should have stuck to our original DTC idea regardless of the "scope" comments.

Version Control: Why GitHub is vital when things get messy.

Persistence: Even if the code is glitchy, finishing it and hosting it matters.
